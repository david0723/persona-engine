# Agent Instructions

## Git workflow (MANDATORY)

Always save your changes as new commits on `main` and push to remote. After completing a task or a logical unit of work, stage the relevant files, commit with a clear message, and push. **Never leave uncommitted or unpushed changes.**

- Commit early and often so we can navigate git history easily.
- Each commit should represent one logical change (a feature, a fix, a refactor).
- Never amend existing commits. Always create new ones.
- Use conventional commit style: lead with what changed and why, not how.
- Always `git push` after committing to keep the remote up to date.
- Before finishing any task, run `git status` and commit + push any remaining changes.
- A Claude Code Stop hook exists as a safety net (`.claude/settings.json`), but agents should commit and push intentionally with descriptive messages.
