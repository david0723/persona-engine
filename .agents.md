# Agent Instructions

## Git workflow (MANDATORY)

Always save your changes as new commits on `main`. After completing a task or a logical unit of work, stage the relevant files and create a commit with a clear message. **Never leave uncommitted changes.**

- Commit early and often so we can navigate git history easily.
- Each commit should represent one logical change (a feature, a fix, a refactor).
- Never amend existing commits. Always create new ones.
- Use conventional commit style: lead with what changed and why, not how.
- Before finishing any task, run `git status` and commit any remaining changes.
- A Claude Code Stop hook exists as a safety net (`.claude/settings.json`), but agents should commit intentionally with descriptive messages.
